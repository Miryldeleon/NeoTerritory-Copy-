# `AboutPage.tsx`

## Sole job

Thesis-specific narrative. Why this attack on the problem in the first place. Not a job-resume page, not a "meet the team" page.

## Layout

1. **Eyebrow + headline** — "What we set out to prove."
2. **Research question** — one paragraph. The exact research question the thesis answers.
3. **Why existing tools fall short** — three short bullets. One sentence each.
4. **Hypothesis** — one paragraph. The mechanism we believe will work and why.
5. **Method** — one paragraph. How we test the hypothesis (lexical tagging + virtual parse tree + JSON-driven catalog + AI doc layer).
6. **Expected contribution** — one paragraph. What changes if the thesis holds.
7. **Acknowledgements** — one paragraph. Adviser, school, anyone whose feedback shaped the work.
8. **Optional team strip** — small monogram chips with names only. No bios, no photos required, no LinkedIn dump.

## Hard rules (per D40, D41, D42)

- No marketing copy. This page is the most academic surface of the site.
- No mention of `/why`, `/mechanics`, `/patterns`. Each section stands alone.
- One motion effect maximum: none. Pure static.

## Route

`/about`. In top nav per D43.
